xObserve 支持两种数据库：`Mysql` 和 `Sqlite`。你可以使用其中任意一种来存储你的数据。

默认情况下，xObserve 使用 `sqlite` 来存储数据。你可以在 xObserve 的根目录下找到一个名为 `xobserve.db` 的文件，该文件就是 sqlite 的数据库存储文件。

<img src={require('@/img/docs/admin/sqlite-file.jpg').default.src} alt="sqlite-file"  className="rounded-lg ring-1 ring-black/5"  style={{height: '100px'}}/>

因此，如果你使用默认的 `sqlite`，请定期备份这个 `xobserve.db` 文件。

## Sqlite 

使用 `sqlite` 非常简单:

<TextList>
    <li>如果你没修改过配置，那么 xObserve 默认就使用 `sqlite` 作为存储</li>
    <li>或者按照如下配置来设置使用 `sqlite`</li>
</TextList>

```yml
# in config.yml
database:
    type: sqlite
```

## Mysql 

与 Sqlite 相比，Mysql 最大的优势自然是存储与应用服务器的分离，这样应用服务器就可以避免单点运行，同时更便于运维管理。

因此，我们推荐大家在生产环境尽量使用 Mysql 来存储数据。

相比 Sqlite，mysql 需要你导入相应的 SQL 文件来创建数据库和表，以及配置好数据库连接信息。

1. 在 `config.yml` 中配置数据库连接信息

```yml
database:
    type: mysql
    account: root
    account_secret: 
    host: localhost
    port: 3306
    database: xobserve
```

2. 将 <Code>xobserve/query</Code> 目录下的 `xobserve.sql` 导入到 Mysql 中。
